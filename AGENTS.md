# AGENTS.md

このリポジトリで利用する自動エージェント（Copilotタスク/エージェント）についての設定と利用方法を示すドキュメントです。

## 目的

- Copilot やチームメンバーがエージェントの役割、利用方法、起動コマンドを理解できるようにする。
- エージェントの名前、責務、入出力、実行方法の簡潔な参照を提供する。

## 配置

- ファイルはリポジトリルートに配置します。Copilot はルートのこのファイルを参照できます。

## 書式方針

- Markdown 見出しで各エージェントを分ける。
- 名前、説明、トリガー（コマンド例）、入力、出力、注意点を明示する。
- 必要に応じて実行例（bash コマンド）を記載する。

エージェント定義はリポジトリ内の .github/agents/*.agent.md に移行しました。各エージェントの詳細（説明・トリガー・入出力・注意点）は該当ファイルを参照してください。

## 運用上の注意

- このドキュメントは人間とCopilotの両方が参照できることを想定してシンプルに保つ。
- エージェントの追加・更新があれば本ファイルを更新すること。

# 開発について

## フェーズ

1. 仕様検討フェーズ: Copilot と対話して問題の調査・要件整理を行う（目的、制約、代替案を明記）。
2. 仕様確定フェーズ: spec/adr に ADR（Architectural Decision Record）を作成して設計判断を記録する（例: spec/adr/2026-04-27-add-auth.md）。
3. ドキュメント作成フェース: ADR を基に spec/ に仕様書を作成し、期待される受け入れ条件とテストケースを明示する。
4. 開発フェーズ: 仕様書の差分（patch/変更点）を code-authoring-agent に渡し、実装案とテストを生成させる。
5. リリースフェーズ: 生成された差分は必ず人間がレビュー・承認したうえでコミット／プッシュする。エージェントによるコミットは原則禁止。

## 注意事項:

- 現在のフェーズが不明のときは、必ず尋ねること。
- ADR と仕様書はリポジトリに保持し、将来の追跡とレビューに備えること。
- 不明点は code-authoring-agent 実行前に解消すること。
- データベースマイグレーション、デプロイ、権限を要する操作は手動で実施すること。
 
- 例:
  - spec/adr/0001-add-auth.md
  - spec/authentication.md
  - 仕様差分を渡す例: `task --agent code-authoring --prompt "Apply spec diff: path/to/diff.patch"`

例: エージェント呼び出しテンプレート
```
# 例（ローカル）
# explore-agent を使って調査を行う（実装は環境に依存）
# task --agent explore --prompt "リポジトリのDB接続箇所をリストアップしてください。出力はファイル名:行でお願いします。"
```
