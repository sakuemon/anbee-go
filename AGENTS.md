# AGENTS.md

このリポジトリで利用する自動エージェント（Copilotタスク/エージェント）についての設定と利用方法を示すドキュメントです。

目的
- Copilot やチームメンバーがエージェントの役割、利用方法、起動コマンドを理解できるようにする。
- エージェントの名前、責務、入出力、実行方法の簡潔な参照を提供する。

配置
- ファイルはリポジトリルートに配置します。Copilot はルートのこのファイルを参照できます。

書式方針（読みやすさ重視）
- Markdown 見出しで各エージェントを分ける。
- 名前、説明、トリガー（コマンド例）、入力、出力、注意点を明示する。
- 必要に応じて実行例（bash コマンド）を記載する。

エージェント一覧（推奨テンプレート）

## explore-agent
- 説明: 大規模な調査・並列探索タスク向け。コードベースの横断調査に利用する。
- トリガー例: `task --agent explore --prompt "..."`（実際の運用に合わせて）
- 入力: 調査対象・質問・制約
- 出力: 調査結果の要約、参照ファイル一覧、推奨アクション
- 注意: 状態は持たない。完全な文脈を与えてください。

## task-agent
- 説明: 単一の長時間実行タスク（ビルド、テスト、CI実行など）を行う。
- トリガー例: `task --agent task --prompt "Run tests"`
- 入力: 実行命令、環境設定
- 出力: 実行ログ、成功/失敗状態、アーティファクトへの参照

## code-review-agent
- 説明: コードのレビューを行う。重大な問題（バグ、セキュリティ）に焦点を当てる。
- トリガー例: `task --agent code-review --prompt "Review changes"`
- 入力: 差分や変更ファイルのセット
- 出力: 検出された重要な問題、修正提案
- 注意: スタイルや細かなフォーマットには言及しないポリシーを推奨

## code-authoring-agent
- 説明: 実装・コード変更（小～中規模）を自動で作成・編集するエージェント。機能追加、バグ修正、リファクタリング、テスト追加を想定。
- トリガー例: `task --agent code-authoring --prompt "Implement X with tests: ..."`
- 入力: 明確なタスク記述（目的、要件、制約）、対象ファイル/パッケージ、期待するテストケース
- 出力: 変更されたファイル一覧、差分（patch/PR形式）、実行したテスト結果、コマンドログ
- 注意:
  - コミットやプッシュはデフォルトで自動実行せず、人間の承認を要することを強く推奨する。
  - データベースマイグレーションやデプロイなど権限を伴う操作は行わない。
  - 出力は適用可能なパッチ（git diff/patch）やPR形式を推奨し、設計上の重要判断は必ず人間レビューを要求する。
  - 機密情報（鍵・トークン）は扱わせない。

ベストプラクティス
- エージェントに渡すプロンプトは具体的に。期待する出力形式（例: JSON, YAML, 箇条書き）を明示する。
- 機密情報（トークン、鍵）はエージェントに渡さない。必要ならCIの安全な仕組みを使う。
- 長い調査時はスコープを分割して、部分ごとに実行する。

運用上の注意
- このドキュメントは人間とCopilotの両方が参照できることを想定してシンプルに保つ。
- エージェントの追加・更新があれば本ファイルを更新すること。

## 開発ワークフロー
- 概要: 開発は Copilot との対話（調査／計画）を起点に行い、spec/adr に ADR を作成、続けて spec/ に仕様書を作成し、その仕様差分を code-authoring-agent に渡して実装を行う順序を採用します。
- 標準手順:
  1. Copilot と対話して問題の調査・要件整理を行う（目的、制約、代替案を明記）。
  2. spec/adr に ADR（Architectural Decision Record）を作成して設計判断を記録する（例: spec/adr/2026-04-27-add-auth.md）。
  3. ADR を基に spec/ に仕様書を作成し、期待される受け入れ条件とテストケースを明示する。
  4. 仕様書の差分（patch/変更点）を code-authoring-agent に渡し、実装案とテストを生成させる。
  5. 生成された差分は必ず人間がレビュー・承認したうえでコミット／プッシュする。自動コミットは原則禁止。
- 注意事項:
  - ADR と仕様書はリポジトリに保持し、将来の追跡とレビューに備えること。
  - 不明点は code-authoring-agent 実行前に解消すること。
  - データベースマイグレーション、デプロイ、権限を要する操作は手動で実施すること。
- 例:
  - spec/adr/0001-add-auth.md
  - spec/authentication.md
  - 仕様差分を渡す例: `task --agent code-authoring --prompt "Apply spec diff: path/to/diff.patch"`

例: エージェント呼び出しテンプレート
```
# 例（ローカル）
# explore-agent を使って調査を行う（実装は環境に依存）
# task --agent explore --prompt "リポジトリのDB接続箇所をリストアップしてください。出力はファイル名:行でお願いします。"
```
